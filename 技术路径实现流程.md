## 一、项目背景与目标

### 1.1 研究背景
本项目基于认知障碍研究数据，利用fMRI提取的**247个脑区指标**，通过高斯过程回归（GPR）模型预测"脑龄"，并评估不同特征组合下的模型表现。

### 1.2 人群分类体系
| 分类 | 全称 | 说明 |
|------|------|------|
| NC | Normal Control | 正常认知 |
| SCD | Subjective Cognitive Decline | 主观认知下降（过渡阶段） |
| SCS | (会议中提及的另一过渡阶段组) | 过渡阶段 |
| MCI | Mild Cognitive Impairment | 轻度认知障碍 |
| AD | Alzheimer's Disease | 阿尔茨海默病 |

### 1.3 核心目标
1. **特征重要性评估**：通过多种方法确定最核心的预测特征
2. **模型优化**：对比不同特征组合（210/37/247个特征）的模型性能
3. **跨组泛化验证**：NC人群训练模型，预测SCD/SCS人群脑龄

## 二、当前技术困境

### 2.1 反直觉现象
- **247个全特征**：Pearson R ≈ 0.8+
- **精简至210个**（去除37个"特殊指标"）：R 降至 0.6+
- **关键疑问**：被删除的37个特征可能包含关键预测信息

### 2.2 三种建模方法对比

| 方法 | 脚本文件 | 流程 | 优点 | 缺点 |
|------|----------|------|------|------|
| **方法1** | `PCAt_to_GPR.py` | 原始特征 → PCA降维(95%方差) → 86个PC → GPR | 降维去噪 | 无法评估单个特征重要性 |
| **方法2** | `PCA_GPR.py` | 原始特征 → PCA载荷筛选 → Top 50%特征 → GPR | 可控的特征筛选 | R值偏低，可能保留不足 |
| **方法3** | `GPR.py` | 原始特征直接输入GPR | 保留全部信息 | R=0.63，存在噪声问题 |

### 2.3 特征重要性评估困境
- **删除特征法** vs **打乱特征值法**：两种方法得出的特征排名**不一致**
- 特殊脑区（如血管流速指标）贡献比例大，但去除后模型性能反而下降

### 阶段二：三种特征组合的模型对比（优先级：高）

#### 任务2.1：全特征基准模型（247个特征）
**使用脚本**：`GPR.py`（方法3：原始特征直接输入）

**操作步骤**：
1. 确认`FILE_PATH`指向筛选后的数据文件
2. 运行脚本，记录以下指标：
   - Pearson r
   - R²
   - MAE（平均绝对误差，单位：年）
   - RMSE
3. 保存输出文件：
   - `GPR_cv_predictions.csv`（交叉验证预测结果）
   - `GPR_scatter.png`（预测年龄vs实际年龄散点图）
   - `GPR_metrics.csv`（综合指标）

**预期结果**：R ≈ 0.8+

---

#### 任务2.2：精简特征模型（210个特征）
**使用脚本**：需修改`GPR.py`或创建变体版本

**操作步骤**：
1. 根据会议讨论，确定需要排除的37个"特殊指标"
   - 会议提及可能包括：血管流速、大写开头的特殊指标等
2. 在`load_dataset()`函数中添加特征筛选逻辑：
   ```python

   ```
3. 运行脚本并记录指标

**预期结果**：R ≈ 0.6+（与全特征对比，验证性能下降）

### 阶段三：特征重要性评估（优先级：中）

#### 任务3.1：置换重要性计算（Permutation Importance）
**使用脚本**：`features_importance.py`

**前提条件**：
- 已有训练好的模型文件（`.joblib`格式）
- 确认`PCAt_to_GPR_model.joblib`存在

**操作步骤**：
1. 确认`FILE_PATH`指向正确数据
2. 确认模型路径指向已训练的模型文件
3. 运行脚本
4. 查看输出：`NC_feature_importance.csv`

**输出解读**：
- `importance_mean`：特征重要性均值（越高越重要）
- `importance_std`：重要性标准差（评估稳定性）

---

#### 任务3.2：留一特征消融实验（NC组内）
**使用脚本**：`features_importance_out.py`

**操作步骤**：
1. 确认使用筛选后的数据（45岁以上）
2. 运行脚本
3. 查看输出：`NC_all_feature_ablation_results.csv`

**关键指标**：
- `delta_MAE`：删除该特征后MAE的变化（正值表示该特征有助于降低误差）
- `delta_R2`：删除该特征后R²的变化
- 按`delta_MAE`降序排序，识别最关键特征

**注意**：该脚本计算量较大，每个特征删除后需重训模型，请预留充足时间

---

#### 任务3.3：跨组消融实验（NC→SCD/SCS）
**使用脚本**：
- `feature_importance_out_SCD.py`（NC训练 → SCD预测）
- `feature_importance_out_SCS.py`（NC训练 → SCS预测）

**前提条件**：
- 确保`sixhos_SCD.xlsx`和`sixhos_SCS.xlsx`已放入Yellow文件夹

**操作步骤**：
1. 运行`feature_importance_out_SCD.py`
   - 输出：`SCD_feature_ablation.csv`
2. 运行`feature_importance_out_SCS.py`
   - 输出：`SCS_feature_ablation.csv`

**关键指标**：
- `delta_MAE_SCD` / `delta_MAE_SCS`：跨组预测性能变化
- 识别在跨组泛化中起关键作用的特征

## 七、附录

### 7.1 脚本依赖关系
```
数据准备
    ↓
GPR.py / PCA_GPR.py / PCAt_to_GPR.py（三选一或全部运行）
    ↓
features_importance.py（需要已训练模型）
    ↓
features_importance_out.py（独立运行）
    ↓
feature_importance_out_SCD.py / feature_importance_out_SCS.py（需要SCD/SCS数据）
```

### 7.2 关键术语
| 术语 | 说明 |
|------|------|
| Brain-PAD | Brain Predicted Age Difference，脑年龄差距（预测年龄 - 实际年龄） |
| GPR | Gaussian Process Regression，高斯过程回归 |
| PCA | Principal Component Analysis，主成分分析 |
| Communality | 共同度，反映特征对各PC的综合贡献 |
| Loading | 载荷，特征与主成分的相关性 |
| Permutation Importance | 置换重要性，通过打乱特征值评估重要性 |
| Ablation Study | 消融实验，逐一删除特征评估影响 |
